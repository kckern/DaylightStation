2. Backend — Strategy Pipeline

SuggestionOrchestrator
• Receives gridSize from request
• Runs strategies in priority order
• Enforces per-type max limits
• Fills remaining slots with discovery
• Returns unified sorted array
Strategies (priority order)
1. NextUpStrategy — max 4, lookback configurable
2. ResumeStrategy — partial playhead + Resumable label
3. FavoriteStrategy — from config, show-level
4. MemorableStrategy — pluggable ranker (suffer score)
5. DiscoveryStrategy — lapsed weighted + random, fills remaining
Each strategy implements: async suggest(context, remainingSlots) → Suggestion[]
Context contains: recent sessions, fitness config, household ID, content adapter

3. Config — fitness.yml additions

# New section in fitness.yml
suggestions:
  grid_size: 8                # total cards to fill (2 rows × ~4)
  lookback_days: 10            # how far back for "recent" programs
  next_up_max: 4              # hard cap on next-up cards
  memorable_lookback_days: 90  # window for memorable episodes
  memorable_max: 2            # max memorable cards
  discovery_lapsed_days: 30    # "haven't done in X days" threshold
  discovery_lapsed_weight: 0.7 # 70% lapsed, 30% true random
  favorites:                   # evergreen shows
    - 12345                  # Video Game Cycling show ID

5. Grid Fill Algorithm

// Priority order — earlier strategies get first claim on slots
1. NextUpStrategy → fills up to min(available, 4)
2. ResumeStrategy → fills up to min(available, remaining)
3. FavoriteStrategy → fills up to min(configured, remaining)
4. MemorableStrategy → fills up to min(2, remaining)
5. DiscoveryStrategy → fills all remaining slots

// Dedup: no show appears twice (next_up and favorite for same show → keep next_up)
// If a favorite show also has a next_up episode, the next_up wins and favorite is skipped
