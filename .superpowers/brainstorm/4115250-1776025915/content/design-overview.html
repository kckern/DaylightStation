Design: Cast Target in Header

Section 1 of 4 — Overall flow and component map

The Flow

1. Set Target

Tap cast icon in header → dropdown panel opens → pick device → set shader & volume → panel closes

→
2. Browse

Header now shows active device chip. Browse content as usual — search, playlists, detail views

→
3. Cast

Tap cast button on any content → quick per-cast toggles (shuffle/repeat) → fires to device with target settings

Component Map

MediaApp (existing)
├─ Header (existing — search bar, nav)
├─ CastTargetChip ← NEW — device icon+name, tap to open panel
└─ CastTargetPanel ← NEW — dropdown with device list + shader/volume
├─ ContentBrowser (existing)
└─ CastButton ← MODIFIED — reads target context, shows per-cast toggles
├─ ContentDetailView (existing)
└─ CastButton ← same modification
└─ useCastTarget() ← NEW hook — manages target device + settings in context

State Shape

castTarget: {
  device: {
    id: 'livingroom-tv',
    name: 'Living Room TV',
    type: 'shield-tv',
    capabilities: { contentControl, deviceControl, osControl }
  },
  settings: {
    shader: 'night',    // device-level default
    volume: 40          // device-level default
  }
} | null              // null = no target set
