Design: Progress & Feedback

Section 4 of 4 — What happens after you tap "Cast Now"

The Problem

Wake-and-load takes 10-80 seconds depending on device state (cold boot vs. already on). The user needs to know it's working and which step it's on — otherwise they'll tap again or think it failed.

Progress via Header Chip

The cast target chip in the header transitions through states. No modal, no toast — the chip itself is the progress indicator. You can keep browsing while it works.

How It Works Under the Hood

// 1. CastButton calls POST /api/v1/device/{id}/load?queue=...&shader=...&volume=...

// 3. WakeAndLoadService emits wake-progress events per step
// 4. CastTargetChip reads progress from context → updates label
// 5. On success/error → chip transitions to final state
