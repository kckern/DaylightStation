# Piano curriculum — metadata layer + content categories

**Date:** 2026-07-09
**Program:** `plex:676490` "Piano With Jonny" (Plex section 17, Lectures)
**Status:** Design approved — ready for implementation plan
**Goal (user):** enrich `plex:676490`; build a YAML/metadata adapter for Plex; update the curriculum UX for course grouping, the 3 content categories, and genre/style/difficulty facets.

---

## Problem

The subcourses UX treats every season as one kind of gated, sequential curriculum. But Piano With Jonny is **three fundamentally different kinds of content**, and the rich per-item metadata needed to tell them apart (style, difficulty, course grouping, instructor) is **not available from Plex**:

1. **Plex isn't ingesting the NFOs.** Section 17 (agent `none`) returns date titles and empty genre after a rescan, even though files are correctly named and NFOs are complete.
2. **Plex's TV model is show-level for genre/tags** — episodes can't natively carry per-song style/difficulty even if pushed via API.
3. **The daylight-station container can't read the bulk media** (`/media/kckern/Media` isn't mounted; Plex sees it at `/data/media/...`). So the adapter can't read sidecar `.nfo`s directly.

The authoritative metadata lives only in the on-disk NFOs, reachable from the host but not from the app.

## Approach — Path C: NFO → index → adapter merge

A **host-side generator reads the NFOs and writes a compact metadata index into the app's own data volume**; the **Plex adapter merges that index onto Plex items**, joining on `(season, episode)` — which is reliable because Plex's episode *index* is intact (contiguous `1..N`, matches the NFO `<episode>`) even though its titles/genre are broken. Plex keeps doing what it's good at (tree structure, ratingKeys, streaming); the index supplies the curriculum metadata. The adapter surfaces it natively on `list`/`info`/`playable`, so all consumers get it.

### Content categories (authored, per season)
- **lesson** — structured instruction, sequential within a course (seasons 01–09).
- **reference** — always-on resource material, no gating/progress, pinned to a shelf (season 00; Scales/Exercises are candidates — authored, adjustable in the index).
- **repertoire** — toward playing a piece; browsed by song with facets, not gated (seasons 10 Song Tutorials `kind: tutorial`, 11 Challenges `kind: challenge`, 12 Accompaniments `kind: accompaniment`).

### Grouping
Course/song grouping is by the **`course`** field (the NFO `Course:` tag), NOT the episode number — the resequence to contiguous `1..N` means `index // 100` is dead (and would have capped at 99 vs. Song Tutorials' 182 songs). A course = a contiguous run of episodes sharing a `course` value, ordered by episode index.

### Facets (repertoire)
All authored/present in the index: **difficulty** (`skill`: Beginner/Intermediate/Advanced), **style** (17 values: Blues, Boogie, Classical, Contemporary, Disney, Funk, Gospel, Holiday, Jazz Ballads, Jazz Swing, Latin Jazz, Pop, Ragtime, Rock, Smooth Jazz, Stride, Fundamentals), **instructor** (`credits`), and **kind** (tutorial/challenge/accompaniment). Search by song title.

## Data model

### The index file (in the app data volume)
Path: `data/household/media/curriculum/676490.json` (JSON for size/parse; 2,434 entries). Generated by the CLI; read + cached by the adapter.

```jsonc
{
  "show": 676490,
  "seasons": {
    "0":  { "title": "Reference",       "category": "reference", "pinned": true },
    "1":  { "title": "Pop Soloing",     "category": "lesson", "sequential": true },
    "10": { "title": "Song Tutorials",  "category": "repertoire", "kind": "tutorial",
            "facets": ["difficulty","instructor","style"] }
  },
  "episodes": {
    "10:1": { "title": "Ain’t Misbehavin’ – 1 – Intro", "course": "Ain’t Misbehavin’ – 1",
              "style": "Jazz Ballads", "skill": "Beginner", "instructor": "John Proulx",
              "focus": ["Songs"], "type": "Course", "plot": "Intro. …" }
  }
}
```
Key `"<season>:<episode>"`; adapter joins on Plex `parentIndex:index`.

### The `piano:` namespace (surfaced by the adapter)
The adapter attaches a namespaced `piano` object to each item (season + episode), so it never collides with native Plex fields (`category`, `genre`, `label` all exist natively):
- Season: `piano = { category, kind?, facets?, sequential?, pinned? }`
- Episode: `piano = { course, style, skill, instructor, focus, type, title, plot }` (episode-level authored overrides win over the index defaults if present).

Category authoring lives in the index (Path C). Optionally, season descriptions in Plex can *also* carry a ```yaml piano:``` fence (API-set summaries stick) for human visibility — but the index is the source of truth.

## Adapter (1_adapters/content/media/plex)
- A small **CurriculumIndex loader** (cached, mtime-keyed) reads `data/household/media/curriculum/<show>.json`.
- In `_toListableItem`/`_toPlayableItem` (and season shaping), when the item belongs to an indexed show, **merge** the index entry: set `title`/`summary` from the index (fixing Plex's date titles), and attach `piano` (episode fields) / season `piano` (category block). Native Plex fields untouched; the merge is additive.
- Surfaced through `list`/`info`/`playable` so the piano kiosk (and any consumer) reads `item.piano.*` and a correct `title` natively.

## UX — three lanes (PianoKiosk)
Entering a program shows **Lessons / Reference / Repertoire** lanes + a **Continue** hero, with the persistent Reference shelf reachable mid-lesson.
- **Lessons** → the Season→Course→Lesson drill (grouping by `course`; gated when `sequential`; progress + Continue).
- **Reference** → flat always-on shelf (no gating/progress/denominators).
- **Repertoire** → a **faceted song browser** unifying seasons 10/11/12: search by title; facet chips for difficulty / style / instructor / kind; a song → its parts as an un-gated list; per-song "learned" marker (not curriculum progress).
- `partitionCourses` groups by `course`; the category comes from `season.piano.category`.
