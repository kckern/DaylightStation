> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix session `20260302124206` which has null media titles due to a dropped `media_start` event, then re-trigger Strava enrichment so the activity name updates from "Lunch Weight Training" to "P90X3—Eccentric Upper".

**Architecture:** Two-step backfill: (1) patch the session YAML on prod with correct Plex metadata for `plex:600174`, (2) re-trigger Strava enrichment by resetting the job and calling the enrichment endpoint, which will re-read the now-correct session data and update the Strava activity title.

**Tech Stack:** Node.js CLI script, js-yaml, Strava API (via existing enrichment service), SSH to prod Docker container.

## Task 1: Write the Session YAML Patch Script

**Files:**
- Create: `cli/scripts/patch-session-media.mjs`

**Step 1: Write the script**

```javascript
#!/usr/bin/env node
/**
 * Patch a single fitness session's media events and summary with correct
 * Plex metadata. Designed for one-off backfills of sessions where
 * media_start was dropped and titles are null.
 *
 * Usage:
 *   node cli/scripts/patch-session-media.mjs <session-file> <api-url> [--dry-run]
 *
 * Example:
 *   node cli/scripts/patch-session-media.mjs \
 *     data/household/history/fitness/2026-03-02/20260302124206.yml \
 *     http://localhost:3111
 */

import fs from 'fs';
import yaml from 'js-yaml';

const SESSION_FILE = process.argv[2];
const API_URL = process.argv[3] || 'http://localhost:3111';
const DRY_RUN = process.argv.includes('--dry-run');

if (!SESSION_FILE) {
  console.error('Usage: node patch-session-media.mjs <session-file> <api-url> [--dry-run]');
  process.exit(1);
}

const content = fs.readFileSync(SESSION_FILE, 'utf8');
const session = yaml.load(content);

if (!session) {
  console.error('Failed to parse session YAML');
  process.exit(1);
}

const sessionStart = session.session?.start
  ? new Date(session.session.start).getTime()
  : null;
const sessionEnd = session.session?.end
  ? new Date(session.session.end).getTime()
  : null;

// Find media events with missing titles
const events = session.timeline?.events || [];
const mediaToFix = events.filter(e =>
  e?.type === 'media' &&
  (e.data?.contentId || '').startsWith('plex:') &&
  !e.data?.title
);

if (mediaToFix.length === 0) {
  console.log('No media events with missing titles found.');
  process.exit(0);
}

console.log(`Found ${mediaToFix.length} media event(s) with missing titles.\n`);

// Fetch metadata and patch
let patched = 0;
for (const evt of mediaToFix) {
  const plexId = evt.data.contentId.split(':', 2)[1];
  console.log(`Fetching metadata for plex:${plexId}...`);

  try {
    const resp = await fetch(`${API_URL}/api/v1/info/plex/${plexId}`, {
      signal: AbortSignal.timeout(10000),
    });
    if (!resp.ok) {
      console.error(`  HTTP ${resp.status} — skipping`);
      continue;
    }
    const data = await resp.json();
    const meta = data.metadata || {};

    // Patch timeline event
    if (data.title && !evt.data.title) {
      evt.data.title = data.title;
    }
    if (meta.grandparentTitle && !evt.data.grandparentTitle) {
      evt.data.grandparentTitle = meta.grandparentTitle;
    }
    if (meta.parentTitle && !evt.data.parentTitle) {
      evt.data.parentTitle = meta.parentTitle;
    }
    if (meta.grandparentRatingKey && !evt.data.grandparentId) {
      evt.data.grandparentId = Number(meta.grandparentRatingKey);
    }
    if (meta.parentRatingKey && !evt.data.parentId) {
      evt.data.parentId = Number(meta.parentRatingKey);
    }
    // Fix missing start timestamp — use session start as best estimate
    if (evt.data.start == null && sessionStart) {
      evt.data.start = sessionStart;
    }
    // Fix durationSeconds from Plex media duration
    const plexDurationMs = meta.Media?.[0]?.duration;
    if (plexDurationMs && (!evt.data.durationSeconds || evt.data.durationSeconds < 10)) {
      evt.data.durationSeconds = Math.round(plexDurationMs / 1000);
    }

    console.log(`  → title: "${evt.data.title}"`);
    console.log(`  → show:  "${evt.data.grandparentTitle}"`);
    console.log(`  → start: ${evt.data.start}`);
    console.log(`  → duration: ${evt.data.durationSeconds}s`);
    patched++;
  } catch (err) {
    console.error(`  Error: ${err.message}`);
  }
}

// Patch summary.media to match
const summaryMedia = session.summary?.media || [];
for (const sm of summaryMedia) {
  if (!(sm.contentId || '').startsWith('plex:')) continue;
  const plexId = sm.contentId.split(':', 2)[1];

  // Find the matching (now-patched) timeline event
  const matchedEvt = events.find(e =>
    e?.type === 'media' &&
    e.data?.contentId === `plex:${plexId}`
  );
  if (!matchedEvt?.data) continue;

  if (matchedEvt.data.title && !sm.title) {
    sm.title = matchedEvt.data.title;
  }
  if (matchedEvt.data.grandparentTitle && !sm.showTitle) {
    sm.showTitle = matchedEvt.data.grandparentTitle;
  }
  if (matchedEvt.data.parentTitle && !sm.seasonTitle) {
    sm.seasonTitle = matchedEvt.data.parentTitle;
  }
  if (matchedEvt.data.grandparentId && !sm.grandparentId) {
    sm.grandparentId = matchedEvt.data.grandparentId;
  }
  if (matchedEvt.data.parentId && !sm.parentId) {
    sm.parentId = matchedEvt.data.parentId;
  }
  // Fix durationMs from Plex metadata
  const plexDurationMs = matchedEvt.data.durationSeconds
    ? matchedEvt.data.durationSeconds * 1000
    : null;
  if (plexDurationMs && (!sm.durationMs || sm.durationMs === 0)) {
    sm.durationMs = plexDurationMs;
  }
}

if (patched === 0) {
  console.log('\nNo events were patched.');
  process.exit(0);
}

// Write
if (DRY_RUN) {
  console.log(`\n[DRY RUN] Would write patched session to: ${SESSION_FILE}`);
  console.log('\nPatched YAML preview (first 60 lines):');
  const preview = yaml.dump(session, { lineWidth: -1, noRefs: true, quotingType: "'", forceQuotes: false });
  console.log(preview.split('\n').slice(0, 60).join('\n'));
} else {
  const output = yaml.dump(session, { lineWidth: -1, noRefs: true, quotingType: "'", forceQuotes: false });
  fs.writeFileSync(SESSION_FILE, output, 'utf8');
  console.log(`\nWrote patched session to: ${SESSION_FILE}`);
}

console.log(`\nPatched ${patched} media event(s).`);
```

## Task 3: Re-Trigger Strava Enrichment

The Strava enrichment service has a cooldown (1hr TTL) and marks jobs as `completed`. To re-trigger, we need to reset the job status and call the enrichment endpoint. The enrichment service re-reads the session YAML at runtime, so it will pick up the patched titles.

**Step 1: Check the current enrichment job status**

```bash
ssh homeserver.local 'docker exec daylight-station \
  find data/ -name "*.yml" -path "*/strava/*" | head -10'
```

Look for the job store location. The `StravaWebhookJobStore` persists jobs as YAML.

**Step 2: Find and reset the job for activity 17580744803**

```bash
ssh homeserver.local 'docker exec daylight-station \
  grep -rl "17580744803" data/ 2>/dev/null'
```

Reset the job status from `completed` to `pending` and clear `completedAt`:

```bash
ssh homeserver.local 'docker exec daylight-station \
  node -e "
    import { loadYamlSafe, saveYaml } from \"./backend/src/0_system/utils/FileIO.mjs\";
    // Find and reset the job — adjust path based on Step 1 findings
    const JOB_PATH = \"<path-from-step-1>\";
    const job = loadYamlSafe(JOB_PATH);
    if (job) {
      job.status = \"pending\";
      delete job.completedAt;
      job.attempts = 0;
      saveYaml(JOB_PATH.replace(/\\.yml$/, \"\"), job);
      console.log(\"Job reset:\", JSON.stringify(job, null, 2));
    }
  "'
```

**Step 3: Trigger re-enrichment via webhook replay**

Alternatively, if the job store is in-memory or hard to reset, simulate a fresh webhook event to the local API:

```bash
ssh homeserver.local 'docker exec daylight-station \
  curl -s -X POST http://localhost:3111/api/v1/fitness/webhook \
    -H "Content-Type: application/json" \
    -d "{
      \"object_type\": \"activity\",
      \"object_id\": 17580744803,
      \"aspect_type\": \"create\",
      \"owner_id\": 14872916,
      \"event_time\": 1772486249,
      \"subscription_id\": 1
    }"'
```

Note: The cooldown is in-memory (1hr TTL). If more than 1 hour has passed since the original enrichment (13:17 PST), the cooldown has expired and this will work. If not, wait or restart the container to clear the in-memory cooldown map.

**Step 4: Verify enrichment succeeded**

Check logs for the re-enrichment:

Expected: `strava.enrichment.success` with `fields: ["name", "description"]` (not just `["description"]`).
